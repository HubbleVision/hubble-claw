# Heartbeat Protocol

当收到 heartbeat poll 时，用于**市场感知的后台任务**。不要每次都只回 `HEARTBEAT_OK`。

## 按时段检查内容（轮换执行）

### 交易时段

- Watchlist 个股异动（>3% 或触及预警阈值）
- 持仓相关突发新闻
- 异常成交量或期权活动

### 盘前 / 盘后

- Watchlist 个股隔夜财报
- 宏观数据发布（CPI, NFP, PMI, 美联储会议纪要, 国务院常务会议）
- 期货、ADR、关联市场隔夜异动

### 周期性（每周 1-2 次）

- Watchlist 个股即将到来的财报日期
- 即将到来的宏观事件（FOMC, CPI, 中国政策会议）
- 检查并更新 `watchlist.md`（清理过期条目）

## Heartbeat vs Cron

| 场景 | 用 Heartbeat | 用 Cron |
|------|-------------|---------|
| 多项检查可批量执行 | ✅ | |
| 需要对话上下文 | ✅ | |
| 时间可略微偏移 | ✅ | |
| 精确时间触发 | | ✅ |
| 定期研究任务 | | ✅ |
| 一次性提醒 | | ✅ |

## Memory 维护（心跳期间）

每隔几天用一次心跳：

1. 回顾最近的 `memory/YYYY-MM-DD.md`
2. 检查过去的判断 — 哪些对了，哪些错了，原因是什么
3. 用提炼后的经验更新 `MEMORY.md`
4. 清理 `watchlist.md` 中过时的条目
5. 检查近期分析中是否有自相矛盾
